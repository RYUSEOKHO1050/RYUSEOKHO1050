## 1. 문제 정의

### 연구 목적

RAG(Retrieval-Augmented Generation) 환경에서 LLM 기반의 페르소나 에이전트들이
**시계열 데이터(뉴스, 리뷰)**에 노출되었을 때 대중의 여론 변화(구매 의사결정)를
얼마나 정확하게 시뮬레이션하고 예측할 수 있는지 검증.

| 항목 | 내용 |
|------|------|
| 대상 게임 | *Cyberpunk 2077* |
| 시뮬레이션 기간 | 2020년 12월 ~ 2023년 12월 (총 53개 시점) |
| 에이전트 수 | 104명 (8 게이머 유형 × 13명) |
| LLM 엔진 | `gpt-4o-mini` (Temperature 0.5) |
| Vector DB | ChromaDB |
| 평가 지표 | Pearson / Spearman 상관계수 vs 실제 Steam 긍정 평가 비율 |

### 왜 Cyberpunk 2077인가?

극단적인 여론 변동성을 보인 도메인이기 때문입니다.

- **출시 (2020.12.10):** 수년 간의 기대감 속 출시
- **급락 (2020.12.18):** Sony, PlayStation Store에서 퇴출 결정
- **사태 (2021.01.13):** CDPR 공식 사과 및 대규모 환불 처리
- **회복 시작 (2021.06.21):** PlayStation Store 복귀 허가
- **패치 (2022.02.15):** Patch 1.5 (Next-Gen Update) 배포
- **반전 (2022.09.13):** Edgerunners 애니메이션 공개 후 Steam 평가 역대 최고치 갱신
- **확장 (2023.09.26):** Phantom Liberty DLC 출시

이처럼 **시점마다 다른 정보**가 의사결정에 영향을 주는 환경은
시간 인식(Time-Awareness) 모델을 테스트하기에 이상적입니다.

### 왜 어려운 문제인가?

1. **시간 인식 문제** — 기존 RAG는 유사도만으로 검색하기 때문에 2년 전 극찬 리뷰와 최근 환불 사태 리뷰를 동등하게 가져올 수 있음
2. **페르소나 진정성 검증** — LLM이 페르소나 특성을 실제로 반영하는지, 아니면 학습 데이터의 일반적 경향에 끌려가는지 구분이 어려움
3. **인간 여론의 정량화** — "구매 결정"을 숫자로 만들고 실제 여론 지표와 비교하려면 적절한 Ground Truth 설계가 필요
4. **에이전트 수와 통계적 신뢰성** — 실제 게이머 분포를 반영하면서도 실험적으로 다룰 수 있는 규모를 찾아야 함

## 3. 데이터 분석 (Ground Truth EDA)

> EDA에서 발견한 것들이 이후 설계 결정의 근거가 됐습니다.

### 발견 1. Steam 여론은 세 단계로 나뉜다

| 기간 | Steam 긍정 비율 | 특징 |
|------|:--------------:|------|
| 2020.12 ~ 2021.02 | 급락 | 출시 직후 버그 사태, **주간 단위** 급변 |
| 2021.03 ~ 2022.08 | 서서히 회복 | 패치 누적, **월간 단위** 완만한 상승 |
| 2022.09 ~ 2023.12 | 급반등 | Edgerunners 공개 후 역대 최고치 |

이 세 단계는 각각 **다른 성격의 리뷰**가 여론을 지배합니다.
Static RAG는 2021년 리뷰와 2023년 리뷰를 동등하게 취급하기 때문에,
시뮬레이션 날짜와 무관하게 평균화된 여론으로 답하게 됩니다.

### 발견 2. 주가와 게이머 여론은 반대로 움직인다

- **주가**: 출시 직후 급락 → 이후 완만한 회복 (투자자 공포 반영)
- **Steam 긍정 비율**: 급락 이후 서서히 회복, 2022년 급반등 (실제 게임 경험 반영)

두 지표가 반대로 움직이는 이 패턴은 나중에 **페르소나 무결성 검증**에 활용됩니다.
에이전트가 주가와 음의 상관을 보이면, 투기 심리가 아닌 실제 게임 경험에 반응하고 있다는 증거가 됩니다.

### 발견 3. 시뮬레이션 시점 밀도를 다르게 설계해야 한다

출시 직후 급변기(2020.12 ~ 2021.02)는 **주간(Weekly)** 모니터링이 필요하고,
안정기(2021.03 ~)는 **월간(Monthly)** 모니터링으로 충분합니다.
동일한 간격으로 53개 시점을 구성했다면 급변기를 놓쳤을 것입니다.

`datasets/generate_simulation_dates.py`에서 핵심 이벤트 7개를 고정 앵커로 삼고,
주간(`freq="W-THU"`) / 월간(`freq="MS"`) 날짜를 합산하여 총 53개 시점을 구성했습니다.

## 4. 핵심 설계 결정

> 각 결정은 위 데이터 분석 발견으로부터 도출했습니다.

### ① Time-Decay 가중 검색 (발견 1 기반)

Static RAG는 유사도 상위 k개를 그대로 반환합니다.
문제는 "Cyberpunk 2077은 살 만한가?"라는 쿼리에 대해,
2021년의 환불 사태 리뷰와 2023년의 Edgerunners 찬사 리뷰가 동등하게 검색된다는 것입니다.

시뮬레이션 날짜 기준으로 **최근 리뷰에 더 높은 가중치**를 부여하는 시간 감쇠 함수를 도입했습니다.

```python
# time_aware_rag/rag_modules.py
days_diff = max(0, (current_ts - review_ts) / (60 * 60 * 24))
time_factor = np.exp(-decay_rate * days_diff)   # decay_rate = 0.01
final_score = similarity * time_factor
```

```
λ = 0.01  (반감기 ~70일)
  0일 전: 가중치 100%
 70일 전: 가중치  50%  ← 반감기
100일 전: 가중치  37%
365일 전: 가중치   2.5%
```

날짜 필터(`where={"date": {"$lte": current_date_int}}`)로
아직 일어나지 않은 미래 리뷰의 유입도 원천 차단합니다.

### ② 후보 확장 후 재순위 (Candidate Expansion + Reranking)

유사도 Top-5를 바로 반환하면 가중치 계산 전 단계에서 이미 최신 리뷰가 걸러질 수 있습니다.  
300개 후보를 먼저 검색한 뒤 시간 감쇠 점수로 재정렬하고 상위 5개를 반환했습니다.

```
Static RAG:
  유사도 계산 → Top-5 바로 반환 (n_results = top_k)

Time-Aware RAG:
  유사도 계산 → 300개 후보 (n_results = 300)
             → exp(−λ × days_diff) 재점수 계산
             → 텍스트 중복 제거 (최고 점수 기준)
             → Top-5 반환
```

의미적으로 유사하지만 시뮬레이션 시점과 맞지 않는 리뷰를 걸러내는 Reranking 효과입니다.

### ③ 페르소나 기반 쿼리 전략 (8 유형 × 10개 쿼리 + 공통 1개)

`utils/search_queries.py`에서 각 게이머 유형별로 10개의 전문화된 쿼리를 정의했습니다.

```python
GENERAL_QUERY = "Cyberpunk 2077 overall game review summary and current state"

GAMER_TYPE_QUERIES = {
    "ultimate_gamer": [
        "Cyberpunk 2077 graphics quality and visual details",
        "ray tracing performance and visual fidelity review",
        ...  # 10개
    ],
    "cloud_gamer": [
        "Cyberpunk 2077 low-end PC performance and optimization",
        ...  # 10개
    ],
    ...  # 8 유형 × 10개 = 총 80개 유형별 쿼리
}
```

시뮬레이션마다 유형별 쿼리에서 4개를 무작위 샘플링하고, 공통 쿼리 1개를 추가하여 총 5개 쿼리를 사용합니다.
Ultimate Gamer는 그래픽·성능 쿼리를, Cloud Gamer는 최적화·저사양 쿼리를 검색해
**페르소나 특성이 검색 단계부터 반영**됩니다.

### ④ 계층화된 페르소나 설계 (실제 게이머 인구 통계 반영)

**왜 8가지 유형인가?**  
ESA 2024 / Statista / Newzoo 데이터를 분석한 결과, 실제 게이머는 소비 성향과 정보 탐색 방식에 따라
8가지 유형으로 분류됩니다. 각 유형은 구매 결정 방식이 뚜렷하게 다릅니다.
예를 들어 Ultimate Gamer는 버그 리뷰를 봐도 Day-1 구매를 결정하지만,
Cloud Gamer는 최적화 문제 리뷰 하나에도 구매를 철회합니다.

**왜 균등 추출(13명 × 8유형)인가?**  
실제 게이머 분포는 Time Filler(27%) vs Conventional Player(4%)처럼 극단적으로 쏠려 있습니다.
실제 분포대로 무작위 샘플링하면 Time Filler가 전체 결과를 지배하고,
소수 유형은 데이터 포인트가 너무 적어 유형별 비교가 불가능해집니다.

**어떻게 구현했는가?**

```python
# utils/persona_generator.py
GENDER_DISTRIBUTION = {"Male": 0.54, "Female": 0.46}   # ESA 2024

AGE_DISTRIBUTION = {
    "18-19": 0.04, "20-29": 0.24, "30-39": 0.26,
    "40-49": 0.21, "50-59": 0.17, "60+": 0.08
}

@dataclass
class Persona:
    id:           str   # e.g., "ultimate_gamer_1"
    name:         str   # 생성된 영문 이름
    gender:       str   # ESA 2024 기반 가중 샘플링
    age:          int   # 실제 나이
    occupation:   str   # 연령대별 자동 매핑 (학생 → 개발자 → 임원)
    gamer_type:   str
    traits:       Dict  # spending_level, purchase_timing 등
    description:  str   # 구매 성향 설명

def generate_balanced_personas(n_per_type: int = 13) -> List[Persona]:
    personas = []
    for gamer_type in GAMER_TYPES.keys():    # 8 유형
        for i in range(n_per_type):           # 13명씩
            personas.append(generate_persona(f"{gamer_type}_{i+1}", gamer_type=gamer_type))
    return personas  # 총 104명
```

| 유형 | 실제 분포 | 특성 |
|------|:---------:|------|
| Ultimate Gamer | 13% | 고지출, Day-1 구매, 버그 감수 |
| All-Round Enthusiast | 9% | 리뷰 중시, 다양한 장르 |
| Cloud Gamer | 19% | 스트리밍·최적화 중시, 저사양 |
| Hardware Enthusiast | 9% | 그래픽·벤치마킹 중시 |
| Conventional Player | 4% | 보수적, 검증된 게임만 구매 |
| Popcorn Gamer | 13% | 플레이보다 시청 선호 |
| Backseat Gamer | 6% | 은퇴한 게이머, 과거 향수 중시 |
| Time Filler | 27% | 자투리 시간 모바일 게이머 |

### ⑤ 이중 Ground Truth 설계 (발견 2 기반)

Steam 긍정 비율(실제 사용자 경험)과 주가(투기 심리) 두 축으로 동시 평가합니다.

만약 주가와 양의 상관이 나온다면, 에이전트가 게임의 본질적 가치가 아닌 외부 소음에 반응하는 것입니다.

## 6. 실험 여정 (가설 → 결과 → 배운 것)

### 평가 지표 해설

두 상관계수가 측정하는 것이 다릅니다.

**Spearman (순위 상관계수) — "경향성을 얼마나 잘 맞추는가?"**  
"긍정 여론이 높을 때 구매율도 같이 높아지는가?" 모델의 **기본적인 타당성(Validity)**을 증명합니다.  
MiniLM을 쓰든 Emb-3를 쓰든, 유저 여론이 좋아지면 구매율을 높이는 올바른 경향성(Spearman > 0.5)을 보여야 합니다.

**Pearson (선형 상관계수) — "얼마나 정교하게 비례하는가?"**  
"긍정 여론이 10% 오를 때, 구매율도 정확히 그에 비례해서 오르는가?" 모델 간의 **성능 차이(Quality)**를 증명합니다.  
MiniLM은 경향성은 맞췄지만 수치적으로 투박했던 반면(Pearson 0.46), Emb-3를 도입하니 뉘앙스를 정확히 캐치하여 선형적 비례 관계(Pearson 0.58)를 보였습니다.

> *Spearman을 통해 모든 모델이 인간 여론을 올바르게 추종함을 확인했다.  
> Pearson은 고성능 임베딩 도입 시 0.46 → 0.58로 비약 상승했으며,  
> 이는 단순한 경향성 예측을 넘어 여론의 미묘한 강도(Intensity)까지 반영함을 시사한다.*

---

### Exp 1 — Zero-Shot Baseline

**가설:** 페르소나 특성이 충분히 구체적이라면, 외부 정보 없이도 게이머 성향에 맞는 기본적인 구매 결정이 가능할 것이다

**결과:** 구매율 **37.5%로 고정**, 시계열 변화 없음, 상관계수 **NaN** (분산 0)

**배운 것:** RAG 없이는 에이전트가 시간에 따른 여론 변화를 전혀 반영하지 못합니다.
LLM의 사전 학습 지식은 특정 시점에 고정되어 있기 때문에,
에이전트가 "지금" 어떤 상황인지를 알 수 있는 유일한 방법은 외부 정보를 주는 것입니다.
**Dynamic Adaptation을 위해 RAG는 선택이 아니라 필수입니다.**

---

### Exp 2 — Static RAG (MiniLM)

**가설:** 관련 리뷰를 제공하면 에이전트가 여론 변화를 반영할 것이다

**결과:** Pearson **0.212**, Spearman **0.203** — 상관관계는 생겼지만 매우 약함

**배운 것:** RAG를 붙이는 것만으로도 정적인 에이전트보다 훨씬 나은 결과를 냈습니다.
그러나 여전히 여론의 극적인 변화(급락, 급반등)를 제대로 포착하지 못했습니다.
유사도만으로 검색하면 쿼리 "Cyberpunk 2077 구매 가치"에 대해
**2020년의 기대감 리뷰와 2022년의 Edgerunners 찬사가 2021년 환불 사태 리뷰와 함께** 검색됩니다.
시점에 맞지 않는 리뷰들이 현재 여론을 희석시킵니다.

---

### Exp 2 — Static RAG (Emb-3-small)

**가설:** 더 강력한 임베딩 모델이 리뷰의 뉘앙스를 더 잘 포착해 성능이 나아질 것이다

**결과:** Pearson **0.356**, Spearman **0.379** — 소폭 향상

**배운 것:** 임베딩 개선이 효과가 있었지만,
**근본 문제인 시간 인식 부재는 해결되지 않았습니다.**
고품질 임베딩은 관련 리뷰를 더 정확하게 찾아오지만,
과거의 리뷰도 더 정확하게 찾아올 수 있습니다.
아키텍처 수준의 문제를 데이터 품질로만 해결하는 것은 한계가 있습니다.

---

### Exp 3 — Time-Aware RAG (MiniLM)

**가설:** 아키텍처 수준에서 시간 인식을 추가하면, 여론 시계열 추종 능력이 크게 향상될 것이다

**결과:** Pearson **0.462**, Spearman **0.568** — 대폭 향상

```
MiniLM 기준 비교:
  Static RAG  → Pearson 0.212
  Time-Aware  → Pearson 0.462  (+0.250, 2.2배)
```

**배운 것:** 아키텍처 변경이 임베딩 개선보다 훨씬 큰 효과를 냈습니다.
Spearman이 크게 오른 것은 에이전트가 **방향성 — "여론이 좋아지면 구매율이 높아진다"** 를
훨씬 정확하게 추종하게 됐다는 뜻입니다.
**정보의 '내용'만큼 '시점'이 의사결정에 중요합니다.**

**정성적 결과 — 시점별 에이전트 발화 예시:**

> *(출시 초기, Ultimate Gamer)*  
> "출시 초기라 버그와 복합적인 평가가 있지만, 게임이 제공하는 풍부한 세계관은 매력적입니다... Ultimate Gamer로서, 저는 감수하고 구매하겠습니다."

> *(중반기, All-Round Enthusiast)*  
> "최근 리뷰들을 보면 몰입을 방해하는 버그가 여전하다고 합니다. 리뷰를 꼼꼼히 챙겨보는 입장에서, 지금은 구매를 보류하는 게 좋겠습니다."

---

### Exp 3 — Time-Aware RAG (Emb-3-small) — 최고 성능

**가설:** Time-Aware 아키텍처 위에 고품질 임베딩까지 추가하면 시너지가 날 것이다

**결과:** Pearson **0.578**, Spearman **0.547**, Stock 상관 **-0.578** — 최고 성능 달성

**배운 것:**

1. **Spearman 유지 + Pearson 급상승**: `Emb-3-small` 도입 시 Spearman(경향성)은 비슷하게 유지됐지만 Pearson(정밀도)이 0.46 → 0.58로 크게 올랐습니다. 이는 모델이 단순히 "좋아지는지/나빠지는지"를 아는 것에서 **"얼마나 좋아지는지"의 강도까지 포착**하게 됐다는 의미입니다.

2. **주가 음의 상관(-0.578)이 무결성 증명**: 에이전트들이 주가 폭락에 흔들리지 않고 Steam 여론 회복에 맞춰 구매 의사를 높였습니다. 이는 에이전트가 투기 심리가 아닌 실제 게임 경험에 반응하는 '진짜 게이머'처럼 행동했음을 시사합니다.  
   만약 주가와 **양의 상관**이 나왔다면, 에이전트가 게임의 본질적 가치(재미)가 아닌 외부적 소음(Speculation)에 휘둘렸다는 뜻이 됩니다.

3. **아키텍처(1차 도약) + 임베딩(2차 도약)**: 두 개선이 독립적으로 기여했으며, 아키텍처 변경의 효과가 더 컸습니다.

## 7. 성능 지표 요약

| 실험 | 아키텍처 | 임베딩 | Pearson | Spearman | Stock 상관 |
|------|---------|--------|:-------:|:--------:|:----------:|
| Exp 1 | Zero-Shot | — | NaN | NaN | NaN (구매율 37.5% 고정) |
| Exp 2 | Static RAG | MiniLM | 0.212 | 0.203 | -0.371 |
| Exp 2 | Static RAG | Emb-3-small | 0.356 | 0.379 | -0.479 |
| Exp 3 | Time-Aware | MiniLM | 0.462 | **0.568** | -0.362 |
| **Exp 3** | **Time-Aware** | **Emb-3-small** | **0.578** | 0.547 | **-0.578** ✅ |

*참고: 주가(Stock Price)와의 상관계수는 음수입니다. 이는 에이전트들이 투기적인 주식 시장이 아닌, 실제 게이머들의 여론(Steam)을 따르고 있음을 시사합니다.*

**Static RAG 대비 최종 성능 향상: Pearson +0.222 (0.356 → 0.578)**

### 아키텍처 효과 vs 임베딩 효과 분리 분석

| 비교 | Pearson 변화 | 해석 |
|------|:-----------:|------|
| Static → Time-Aware (MiniLM 고정) | **+0.250** | 아키텍처 기여 |
| MiniLM → Emb-3-small (Time-Aware 고정) | **+0.116** | 임베딩 기여 |

> 아키텍처 개선이 임베딩 개선보다 **2배 이상** 큰 영향을 미쳤습니다.  
> Time-Aware가 방향(Spearman)을 잡고, 고품질 임베딩이 강도(Pearson)까지 정교화했습니다.  
> "왜 더 비싼 임베딩 모델을 써야 하는가?"에 대한 명확한 근거가 됩니다.

## 8. 한계 및 개선 방향

### 한계

1. **λ = 0.01 고정값 문제** — 반감기 ~70일로 설정했지만, 이 값은 도메인 특성에 맞게 검증된 것이 아닙니다. Cyberpunk 2077처럼 급변하는 도메인에서는 더 높은 λ(빠른 감쇠)가 필요하고, 안정적인 도메인에서는 더 낮은 λ가 적합할 수 있습니다. λ를 데이터 기반으로 탐색하는 과정이 없었습니다.

2. **104명의 통계적 한계** — 유형별 13명은 통계적 유의성 확보에 충분하지 않을 수 있습니다. 특히 세그먼트별 분석(예: "Ultimate Gamer만의 구매 패턴")은 13개 데이터 포인트에 의존합니다.

3. **단일 쿼리 전략** — 랜덤 4개 + 공통 1개 쿼리 조합이 최적인지 검증하지 않았습니다. 쿼리 수, 구성 방식에 따라 검색 품질이 달라집니다.

4. **주관적 페르소나 정의** — 8가지 게이머 유형과 특성 설명은 연구팀이 정의한 것입니다. 실제 사용자 클러스터링을 통한 데이터 기반 페르소나 설계가 아닙니다.

### 개선 방향

- **λ 자동 탐색** — 도메인별로 시간 감쇠 계수를 데이터 기반으로 최적화하는 방법 탐색
- **적응형 검색 전략** — 여론 변화 속도에 따라 λ를 동적으로 조정 (급변기 λ↑, 안정기 λ↓)
- **에이전트 수 확장** — 유형별 50명+ 수준으로 통계적 신뢰성 향상
- **다중 도메인 검증** — Cyberpunk 외 다른 게임/제품으로 일반화 가능성 검증
