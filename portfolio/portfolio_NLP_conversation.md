# 한국어 일상 대화 연결 (Korean Natural Conversation Continuation)

> AI 말평 대회 · 팀 프로젝트 · 2025

---

## 1. 문제 정의

### 과제 설명

한국어 일상 대화에서 **대상 발화**가 주어졌을 때, 그 앞(선행) 또는 뒤(후행)에 올 수 있는
**가장 자연스러운 발화를 3개의 후보 중 선택**하는 Multiple Choice NLP 과제.

| 항목 | 내용 |
|------|------|
| 대상 발화 | "나는 인제 신경이 쓰여 근까" |
| 발화 위치 | 선행문 (이 발화 앞에 뭐가 왔을까?) |
| 발화 1 ✅ | "근데 저 옆집은 왜 저러는지 모르겠어요." |
| 발화 2 ❌ | "저는 신경통이 있을 때는 약을 먹어요." |
| 발화 3 ❌ | "대학에서 신경과학을 전공하셨다면서요?" |

**평가 지표:** Accuracy | **데이터:** Train 600 / Dev 119 / Test 478

### 왜 어려운 문제인가?

1. **구어체·비표준어** — "신경이 쓰여 근까", "거 같애요" 등 표준어 기반 모델이 잘 처리 못하는 표현
2. **소규모 데이터 (600개)** — Transformer 모델을 파인튜닝하기에 매우 적은 양, 과적합 위험 높음
3. **선행문의 구조적 어려움** — 선행문은 가능한 발화 공간이 무한하지만, 후행문은 응답 범위가 제한적
4. **의미 기반 판별** — EDA 결과 정답/오답의 길이 차이가 0.19자로 거의 없음 → 형태가 아닌 **의미 이해**가 핵심

## 3. 데이터 분석 (EDA)

> EDA에서 발견한 것들이 이후 모든 설계 결정의 근거가 됐습니다.

### 발견 1. 발화위치별 정답 패턴이 다르다

| 발화위치 | 발화1 정답 비율 | 발화2 | 발화3 |
|---------|:-----------:|:-----:|:-----:|
| 선행문 | **37.6%** | 35.7% | 26.7% |
| 후행문 | 33.2% | **34.3%** | 32.5% |

선행문은 발화1 선호가 뚜렷하지만, 후행문은 세 선택지가 균등합니다.

### 발견 2. 길이는 정답의 단서가 아니다

| 구분 | 평균 길이 |
|------|:--------:|
| 정답 발화 | 22.13자 |
| 오답 발화 | 22.32자 |

차이가 0.19자. 형태가 아닌 **의미**로 판단해야 합니다.

### 발견 3. 구어체가 강하고, 부정 표현이 의미 대조의 단서다

- "거 같애요", "네잉", "아이가" 등 비표준 구어 표현 다수
- 부정 표현이 포함된 샘플 약 52% → 부정 표현 유무가 의미 대조의 핵심 단서

### 발견 4. 600개는 Transformer 파인튜닝에 너무 적다

- 증강 없이 단순 학습 시 검증 정확도 70%에서 정체
- train/val split에 따라 성능 편차가 크게 발생

## 4. 핵심 설계 결정

> 각 결정은 위 EDA 발견으로부터 도출했습니다.

### ① Pair-wise 이진 분류로 문제 재정의 (발견 2 기반)

3-class softmax는 세 선택지를 동시에 비교해 상대적으로 높은 것을 고릅니다.
문제는 모델이 "절대적으로 자연스러운가"가 아니라 "세 개 중 상대적으로 높은 것"을 학습하게 된다는 점입니다.

Pair-wise 방식은 대상발화와 후보를 1:1로 독립 평가해 각 선택지를 동등한 기준으로 판단합니다.
실제로 사람이 선택지를 검토하는 방식과 동일하며, 학습이 더 안정적입니다.

```
[3-class 방식]
(대상발화, 발화1, 발화2, 발화3) → softmax(3) → 정답 선택

[Pair-wise 방식]  ← 채택
(대상발화 + 발화1) → P(정답) = 0.23
(대상발화 + 발화2) → P(정답) = 0.71  ← 최댓값 선택
(대상발화 + 발화3) → P(정답) = 0.18
```

### ② 특수 토큰으로 감정·부정 정보 명시화 (발견 3 기반)

모델이 스스로 감정·부정 패턴을 학습할 수는 있지만, 600개 소규모 데이터에서는 일반화가 어렵습니다.
EDA에서 확인한 분석 결과를 토큰으로 직접 주입해 모델이 해당 특성에 집중하도록 유도했습니다.

```
# 특수 토큰 없이:
"너무 슬퍼요 </s> 괜찮아질 거예요"

# 특수 토큰 포함:
"[NEG][NEGT] 너무 슬퍼요 </s> [POS][NONEGT] 괜찮아질 거예요"
```

| 토큰 | 의미 |
|------|------|
| `[PREV]` / `[NEXT]` | 발화 위치 |
| `[POS]` / `[NEG]` / `[NEU]` | 감정 |
| `[NEGT]` / `[NONEGT]` | 부정 표현 여부 |

### ③ 선행문·후행문 모델 분리 학습 (발견 1 기반)

발화위치별 정답 분포가 다르기 때문에, 하나의 모델에 섞으면 편향을 학습할 위험이 있습니다.
위치별로 독립적인 모델을 학습해 각 위치의 담화 특성을 따로 반영했습니다.

### ④ klue/roberta-large 선택 (발견 2 기반)

길이가 단서가 아니고 의미 이해가 핵심임을 확인했기 때문에, 한국어 문맥 이해 벤치마크 최상위권 모델을 선택했습니다.
KcBERT는 구어 표현 처리에 강하지만, 600개 소규모 데이터에서는 넓은 문맥의 의미 관계 추론이 강한 roberta-large가 더 유리하다고 판단했습니다.

### ⑤ 역번역 증강 + 5-Fold CV (발견 4 기반)

600개의 데이터 부족 문제를 두 방향으로 접근했습니다.

- **역번역(KO→EN→KO):** 원문 의미를 보존하면서 표현을 다양화해 600개 → 1,319개로 확장 시도. 하지만 구어체 특성이 손실되는 문제가 뒤따랐습니다 (Exp B-2, B-3 참고).
- **5-Fold CV:** train/val split 의존 문제를 해결하고 일반화 성능 확보.

## 5. 시스템 구조도 (Architecture Diagram)

```
┌─────────────────────────────────────────────────────────────┐
│                        INPUT DATA                           │
│  대상발화: "맞아 자식이래서 어쩔 수 없는 거 같애요 이게 진짜지."   │
│  발화위치: 선행문                                            │
│  후보 3개: 발화1 / 발화2 / 발화3                             │

┌─────────────────────────────────────────────────────────────┐
│              PREPROCESSING & TOKENIZATION                   │
│                                                             │

│             PAIR-WISE BINARY CLASSIFICATION                 │
│                                                             │
│  3지선다 → 후보별 3개 독립 이진 분류 샘플로 분해               │
│                                                             │
│  "[NEU][NONEGT] 대상발화 </s> 발화1"  → Score_1             │

│           BIDIRECTIONAL ENSEMBLE (최종 전략)                 │

│  후행문 샘플 → [후행문 모델 5개] + [선행문 모델 5개 역방향]   │
```

## 6. 실험 여정 (가설 → 결과 → 배운 것)

### Baseline — 파이프라인 검증

**가설:** 구어체 특화 모델(KcBERT)로 기본 파이프라인이 작동하는지 확인  
**결과:** 검증 70.00%, 리더보드 75.73  
**배운 것:** 파이프라인 자체는 정상이지만 성능이 낮음 → 모델과 학습 전략 개선 필요

---

### Exp B-1 — RoBERTa + 특수토큰 + Raw Data

**가설:** 강력한 한국어 모델 + 설계된 특수 토큰이면 성능이 크게 오를 것이다  
**결과:** 리더보드 82.22 (Baseline 대비 +6.49%p)  
**배운 것:** 모델 자체의 언어 이해력이 올라가면 성능이 오름. CV가 없어 특정 split에 의존하는 문제 잔존.

---

### Exp B-2 — 역번역 데이터만으로는 안 된다

**가설:** 역번역으로 생성한 데이터가 원본보다 표준화돼 있으니, 오히려 더 깔끔한 학습이 될 것이다  
**결과:** 리더보드 **63.60 — 오히려 Baseline보다 낮음**  
**배운 것:** 역번역 데이터는 구어체를 표준어로 바꿔버린다. 평가 데이터는 원본 구어체 스타일이므로,
역번역 데이터만으로 학습한 모델은 실제 평가 데이터 분포를 전혀 본 적이 없는 상태가 된다.
**원본 데이터 없이 증강 데이터만 쓰는 것은 도메인을 바꾸는 것과 같다.**

---

### Exp B-3 → Exp B-4 — 과적합 발견과 CV 도입

**가설:** 원본 + 역번역 데이터를 합치면 데이터가 늘어 성능이 오를 것이다  
**결과 (Exp B-3):** 검증 정확도 **97.81%**, 리더보드 **81.17** — 숫자가 역전  
**배운 것:** 검증 정확도가 높다고 실전 성능이 좋은 게 아니다. CV 없이 단일 split에서 97%는 신뢰할 수 없는 수치.

**→ 5-Fold CV 도입 후 (Exp B-4):** 검증 90.50%, 리더보드 **85.98** (과적합 해소 확인)

---

### Exp B-5 — 데이터 품질이 양보다 중요하다는 발견

**가설:** 증강 데이터도 CV를 적용하면 괜찮을 것이다. 더 많은 데이터가 더 좋겠지.  
**결과:** 원본만 + CV → 리더보드 **86.19** (증강+CV 85.98보다 높음)  
**배운 것:** 역번역은 "거 같애요", "아이가" 같은 구어체 표현을 표준어로 바꿔버린다.
**데이터를 늘리는 것보다 원본 데이터의 특성을 보존하는 것이 더 중요하다.**

---

### Exp1 — 양방향 앙상블

**아이디어:** 선행문 모델은 "A 앞에 B가 오면 자연스럽다"는 관계를 학습했다.
같은 관계를 뒤집어서 후행문 모델에 입력해도 유사한 관계를 판단할 수 있지 않을까?
추가 학습 없이 기존 10개 모델을 역방향으로도 활용해 더 다양한 시각에서 앙상블.

**결과:** 86.19 → **87.4477 (+1.26%p) — 최고 성능 갱신**  
**배운 것:** 새로운 데이터나 모델 없이도, **추론 방식의 구조적 재설계**만으로 성능을 끌어올릴 수 있다.

---

### Exp2 — 외부 데이터 실패

**가설:** 국립국어원 말뭉치 데이터를 추가하면 일반화 성능이 오를 것이다  
**결과:** 87.4477 → **82.0084 (−5.44%p)**  
**배운 것:** 단순히 관련 도메인이라고 같은 분포를 가지지 않는다.
**도메인 유사성은 주제가 아니라 레이블 기준과 언어 스타일의 일치여야 한다.**

---

### Exp3 — ML 기반 감성 분석 실패

**가설:** 규칙 기반 키워드 매칭보다 KoELECTRA-NSMC 모델이 더 정교한 감성을 잡아낼 것이다  
**결과:** 87.4477 → **86.4017 (−1.05%p)**  
**배운 것:** KoELECTRA-NSMC는 영화 리뷰 도메인 모델이다. 일상 대화의 감성 표현 방식과 다르다.
**복잡한 방법이 항상 우수하지 않다. 도메인 정합성이 먼저다.**

## 7. 성능 지표 요약

| 실험 | 핵심 아이디어 | 리더보드 | 변화 |
|------|-------------|:-------:|:----:|
| Baseline | KcBERT, 원본 데이터 | 75.73 | — |
| Exp B-1 | RoBERTa + 특수토큰 | 82.22 | +6.49 |
| Exp B-2 | 역번역 데이터만 학습 | 63.60 | −12.13 |
| Exp B-3 | 원본+증강 (CV 없음) | 81.17 | 과적합 |
| Exp B-4 | 원본+증강 + 5-Fold CV | 85.98 | +4.81 |
| **Exp B-5** | **원본 + 5-Fold CV** | **86.19** | +0.21 |
| Exp2 | 외부 데이터 추가 | 82.01 | −4.18 |
| Exp3 | ML 기반 감성 토큰 | 86.40 | −1.05 |
| **Exp1** | **양방향 앙상블** | **87.45** | **+1.26** ✅ |

**베이스라인 대비 최종 성능 향상: +11.72%p (75.73 → 87.45)**

### Cross Validation의 효과

| 조건 | Val Acc | 리더보드 |
|------|:-------:|:-------:|
| CV 없음 (증강) | 97.81% | 81.17 |
| CV 적용 (증강) | 90.50% | 85.98 |
| CV 적용 (원본) | 84.24% | **86.19** |

> 검증 정확도 97%가 오히려 과적합 신호였다. CV가 소규모 데이터에서 필수인 이유.

## 8. 한계 및 개선 방향

### 한계

1. **학습 데이터 600개의 근본적 한계** — 특수 토큰 7개를 학습시키기에 표본이 부족했다.
   특수 토큰 효과가 일부 실험에서만 나타나고 일반화가 안 된 원인.

2. **역번역 증강의 한계** — 구어체 특성이 손실된다. 더 나은 증강 방법이 필요했다.
   (예: 구어체 스타일을 유지하는 paraphrase 방식)

3. **Exp3의 도메인 불일치** — 일상 대화 도메인에 맞는 감성 분석 모델이 없어서
   영화 리뷰 기반 모델을 썼는데, 도메인 차이로 오히려 노이즈가 됐다.

### 개선 방향

- 일상 대화 도메인 특화 감성 분석 모델 파인튜닝
- 구어체 특성을 보존하는 증강 방법 탐색 (동의어 치환, 구어체 paraphrase)
- 앙상블 가중치 단순 평균이 아닌 검증 성능 기반 최적화
